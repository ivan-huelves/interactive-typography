### Descripción
Este proyecto consiste en el desarrollo de una aplicación web orientada a la aplicación de filtros gráficos generativos sobre texto en tiempo real. La herramienta permite al usuario introducir cualquier cadena de texto y modificar su apariencia mediante el control de parámetros. El sistema utiliza p5.js para mapear el área del glifo y sustituir los píxeles resultantes por diferentes módulos gráficos (como tramas de puntos, circunferencias o imágenes) o aplicar algoritmos de ruido como el perlin noise. El objetivo es democratizar el acceso a recursos gráficos complejos, proporcionando una herramienta eficiente para diseñadores que deseen experimentar con la programación creativa sin necesidad de escribir código. La propuesta se vincula con *Cumulus Park*, *GT* y *Visual Chronicles of Abya Yala* en su estrategia de construcción del glifo mediante módulos, puesto que, al igual que estos referentes descomponen el carácter en sistemas de partículas o componentes combinatorios, Typer mapea el área textual para reconstruirla a través de tramas y elementos gráficos repetitivos. Por otro lado, la herramienta conecta con *Leon Sans* y *Bit Tripper* en su naturaleza generativa basada en código, donde la programación permite generar variaciones formales y estructurales en tiempo real a partir de un mismo *input*. Asimismo, el proyecto se alinea con *Space Type Generator* como paradigma de herramienta web orientada a la creación de efectos visuales y cinéticos, democratizando el acceso a la experimentación gráfica compleja. Finalmente, la propuesta resuena metodológicamente con los talleres de Anna Carreras, compartiendo la premisa de que el diseñador deja de construir el artefacto final para pasar a programar las reglas que lo generan. La arquitectura de Typer articula el modelo mediante una relación directa basada en parámetros, donde la cadena de texto y los parámetros de control de la interfaz (como la densidad de puntos y la velocidad) actúan como valores de entrada. El sistema emplea una función normalizadora que rasteriza la anatomía del carácter para generar una retícula de puntos de anclaje, estableciendo así la matriz de datos necesaria para que el motor proyecte la trama sobre la superficie del texto. A partir de estos datos, la función significativa ejecuta una lógica de sustitución gráfica, reemplazando los puntos resultantes por diferentes sets visuales (formas geométricas, emojis o imágenes) animados. Este mecanismo permite generar recursos visuales complejos a partir de un *input* textual simple. De este modo, la propuesta recupera la dimensión generativa de Anti-Stress Type Generator, pero con una diferencia sustancial puesto que mientras que aquel operaba mediante una lógica curatorial sin alterar el diseño tipográfico original, aquí el sistema sí interviene directamente en la morfología del glifo para reconstruirlo, validando la capacidad del modelo para sistematizar la experimentación gráfica formal.
